Connect Google Sheets

Complete this one-time setup to connect your knowledge base to a Google Sheet as the database. Articles and users will be read and written there.

1Create a Google Sheet

Go to sheets.google.com, create a new spreadsheet, and add two tabs named exactly Articles and Users.

In row 1 of Articles, paste these headers:

id title desc category author authorEmail date content helpful notHelpful views status

In row 1 of Users, paste these headers:

id name email password role createdAt

Set sharing to Anyone with the link → Viewer so reads work without OAuth.

Paste your Sheet ID (the long string in the URL between /d/ and /edit):

2Get a Google API Key

Go to console.cloud.google.com → APIs & Services → Library → enable Google Sheets API. Then go to Credentials → Create Credentials → API Key.

Restrict the key to the Sheets API for security.

3OAuth Client ID (for writes)

To write data (create/edit articles, manage users) the app needs OAuth. Go to Credentials → Create Credentials → OAuth 2.0 Client ID → Web application. Add your page URL to Authorised JavaScript origins.

Only needed for Admins. Viewers use read-only API key access.

4Create your first Admin

Settings saved in your browser. No server needed.
